# 🤝 Contributing to HELIX

Welcome to the cutting edge of AI orchestration! By contributing to HELIX, you are helping build the first truly adaptive agentic framework.

## 🤯 The Meta-Contribution

Because HELIX features a **Self-Evolving architecture**, the way we handle contributions is unique. 

**The Evolver learns from PRs!** 
When you submit a Pull Request to the `caramaschiHG/helix-loops` repository, our CI/CD pipeline runs the Evolver against your diff. If your PR introduces a better prompting strategy or a more efficient bash script, the Evolver analyzes it and updates the core global templates for the next release.

## 🛠️ How to Contribute

1. Fork the repo (`https://github.com/caramaschiHG/helix-loops`).
2. Create a feature branch (`git checkout -b feature/amazing-new-logic`).
3. Make your changes (keep them focused!).
4. Run local tests (if applicable).
5. Commit your changes (`git commit -m 'feat: added predictive context routing'`).
6. Push to the branch (`git push origin feature/amazing-new-logic`).
7. Open a Pull Request.

## 📜 Code of Conduct

We are building tools for the future. Be respectful, be curious, and leave your ego at the door. 

- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experiences.
- Gracefully accept constructive criticism.
- Focus on what is best for the community.

If you have questions, reach out to @caramaschiHG or open an Issue. Let's build the smartest loops together.
