# 🤝 Contributing to HELIX v2.0

Welcome to the absolute bleeding edge of AI orchestration! By contributing to HELIX, you are helping build a premium, self-adaptive framework.

## 🤯 The Meta-Contribution

Because HELIX features a **Self-Evolving architecture**, the way we handle contributions is unique. 

**The Evolver learns from PRs!** 
When you submit a Pull Request to the `caramaschiHG/helix-loops` repository, our CI/CD pipeline runs the Evolver against your diff. If your PR introduces a better prompting strategy or a more elegant terminal UI element, the Evolver analyzes it and updates the core global templates for the next release.

## 🛠️ How to Contribute

1. Fork the repo (`https://github.com/caramaschiHG/helix-loops`).
2. Create a feature branch (`git checkout -b feature/premium-dashboard`).
3. Make your changes (keep them focused and elegant!).
4. Run local tests.
5. Commit your changes (`git commit -m 'feat: added visual terminal dashboard'`).
6. Push to the branch (`git push origin feature/premium-dashboard`).
7. Open a Pull Request.

## 📜 Code of Conduct

We are building high-end tools for the future. Be respectful, be curious, and leave your ego at the door. 

- Use welcoming, calm, and professional language (matching our UX philosophy).
- Be respectful of differing viewpoints and experiences.
- Gracefully accept constructive criticism.
- Focus on what is best for the community.

If you have questions, reach out to @caramaschiHG or open an Issue. Let's dominate the market together.
